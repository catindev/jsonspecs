# Реализация движка

Этот документ описывает реализацию `@jsonspecs/rules` для Node.js. Он не копирует
контракт поведения и не переопределяет его.

Нормативное поведение задано в
[`jsonspecs/spec` 1.0.0-rc.5](https://github.com/jsonspecs/spec/blob/d7f07976b0fe004584217adf079c57caec102ab3/SPEC_RU.md).
Точный коммит исходного корпуса записан в `tests/conformance/spec-commit.txt`.

## Компиляция

`createEngine()` создаёт неизменяемый реестр встроенных и переданных внешних
операторов. `createEngine()` один раз компилирует их JSON Schema через Ajv. Контракт
должен явно перечислять допустимые имена конфигурации, `inputs` и непосредственных
полей `params`, а также закрывать каждый из этих уровней объекта.

`compileSnapshot()` выполняет следующие фазы:

1. клонирует и проверяет I-JSON-конверт снэпшота;
2. сверяет JCS-хеш `sourceHash`;
3. проверяет формы артефактов и контракты операторов;
4. разбирает переносимые регулярные выражения и проверяет пределы компиляции;
5. строит индексы принадлежности для справочников;
6. проверяет точные ссылки, общий граф управления и достижимость;
7. сообщает об отсутствующих операторах только после исключения независимых ошибок.

Подготовленный результат открывает только формат и идентификатор набора правил.
Исполняемое состояние, функции операторов, индексы и артефакты хранятся в закрытом
`WeakMap`.

## Выполнение

`runPipeline()` проверяет `pipelineId`, `payload` и `context` до выполнения первого
шага. Объекты среды выполнения итеративно проверяются на недопустимые контейнеры,
циклы, разреженные массивы, опасные ключи, неконечные числа и глубину. Затем
`payload` и `context` клонируются и замораживаются.

Внутренние прерывания помечаются через закрытый `WeakSet`. Обработчики проверяют
метку по идентичности. Поэтому даже `Proxy`, который выбрасывает сам себя, не может
вызвать новую ловушку через `instanceof` или чтение полей ошибки. Ошибки во входных
данных и контексте сохраняют разные коды.

Для `payload` и `context` строятся закрытые проекции путей. Сценарии, условия и
вложенные вызовы выполняются через явные стеки, а не стек вызовов JavaScript.
Оператор получает замороженный набор разрешённых значений, но не весь `payload` и
не средство разрешения путей.

## Регулярные выражения и справочники

Переносимый шаблон разбирается во внутреннее синтаксическое дерево и преобразуется
для `re2-wasm`. До вызова RE2 проверяются грамматика, длина шаблона, отдельные
квантификаторы, множитель вложенных ограниченных повторов и развёрнутое число
атомов.

Элементы справочников проверяются и индексируются по типу во время компиляции.
Встроенные операторы используют закрытые индексы `Set`. Внешний оператор по-прежнему
получает неизменяемый нормативный массив элементов.

## Проверка совместимости

`npm run test:conformance` читает закреплённый коммит, проверяет рабочую копию
`jsonspecs/spec`, побайтно сравнивает весь корпус и только затем запускает фикстуры.
Результаты сравниваются как данные JSON без учёта прототипов и классов среды.
Идентификатор отказа сравнивается и тогда, когда ожидается его отсутствие.

Остальные проверки выпуска описаны в [TESTING.md](TESTING.md).
